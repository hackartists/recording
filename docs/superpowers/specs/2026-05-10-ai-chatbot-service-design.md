## 2. 문제 정의

취업준비생은 면접 실력을 키우고 싶지만 다음과 같은 한계에 부딪힌다.

- **연습 상대 부재** — 가족·친구는 직무 맥락을 모르고, 스터디는 시간 조율이 어려움
- **피드백 부재** — 혼자 연습하면 본인의 답변이 잘했는지 못했는지 알 수 없음
- **반복 비용 부담** — 면접 컨설팅·학원은 1회 5만~30만 원으로 부담이 큼

**핵심 통찰:** 취준생에게 가장 부족한 것은 "정답"이 아니라 "연습 횟수와 즉시 피드백"이다.

### Primary Persona

- **이름:** 김취준 (가상의 페르소나)
- **나이/상태:** 25세, 4학년 2학기 / 졸업 직후

- **원하는 것:** 실제 면접관처럼 질문해주고, 내 답변의 약점을 알려줄 누군가

### Secondary Persona

- **이직 준비자 (28~35세):** 경력직 면접 대비, 회사·직무 다양성 필요

## 5. 핵심 기능 (MVP)

MVP에는 **3가지 기능만** 포함한다.

### 5.1 맞춤 질문 생성

- 사용자가 **회사명 + 직무 + 직급 (신입/경력)** 을 입력
- 선택적으로 **자소서·이력서 텍스트** 붙여넣기 가능
- AI가 해당 맥락에 맞는 면접 질문 5~10개를 생성
- 질문 유형: 인성, 직무, 상황 대응 (질문별로 라벨 표시)

### 5.2 답변 & 즉시 피드백

- 사용자가 **텍스트로 답변** 입력 (음성은 v2)
- AI가 다음 4가지 관점으로 피드백 제공:
  1. **구조** — STAR 기법 등 답변 구성이 명확한가
  2. **구체성** — 추상적 표현 vs 구체적 사례·수치
  3. **직무 적합성** — 답변이 해당 직무·회사 맥락과 맞닿아 있는가
  4. **개선 제안** — 한 줄짜리 더 나은 답변 예시
- 피드백은 한 번에 모두 보이지 않고, **카드 형태로 단계적 노출** (압도감 방지)

### 5.3 연습 기록

- 모든 면접 세션이 자동 저장
- 세션 목록에서 회사/직무/날짜로 다시 보기 가능
- **약점 요약:** 누적 피드백을 바탕으로 "당신이 자주 약한 영역" 을 한 줄로 표시
  - 예: "최근 5회 답변에서 '구체적 수치 부족' 피드백이 4회 있었습니다."

## 6. 사용자 시나리오 (Happy Path)

```
[1] 사용자가 웹사이트 접속 → 이메일로 가입 (소셜 로그인 없음)
        ↓
[2] "새 면접 시작" 클릭
        ↓
[3] 회사명 / 직무 / 직급 입력 → (선택) 자소서 붙여넣기
        ↓
[4] AI가 5~10개 질문 생성 → 사용자가 첫 질문 선택
        ↓
[5] 답변 입력 (텍스트)
        ↓
[6] AI 피드백 4가지 카드로 표시 → 다음 질문으로 이동
        ↓
[7] 세션 종료 → 전체 요약 + "오늘의 약점" 한 줄
        ↓
[8] 마이페이지에서 과거 세션 다시 보기 가능
```

소요 시간: 1세션 평균 **15~20분**.

## 7. 화면 구성 (와이어프레임 수준)

MVP는 다음 4개 화면만 존재한다.

1. **랜딩/로그인** — 서비스 소개 + 이메일 가입/로그인
2. **세션 시작** — 회사·직무·자소서 입력 폼
3. **면접 채팅** — 질문/답변/피드백이 흐르는 챗 화면
4. **마이페이지** — 과거 세션 목록 + 약점 요약

> **시각 목업:** 위 4개 화면의 mock HTML은 프로젝트 루트의 [index.html](../../../index.html) 에서 확인할 수 있다. 디자인·문구는 구현 단계에서 변경될 수 있으며, 본 목업은 기획 검토 및 향후 플랜닝 시 참고용이다.

YAGNI 원칙에 따라 다음은 v1에서 명시적으로 제외한다.

- 음성 답변 / 영상 면접 분석

- 소셜 로그인 (Google, Kakao 등)

- 사용자 간 커뮤니티·답변 공유

- AI 모델 선택 옵션 (단일 모델 고정)

## 12. 리스크 & 대응

| 리스크 | 대응 |
|---|---|
| AI 피드백 품질이 일반론에 그칠 우려 | 회사·직무 메타데이터를 프롬프트에 강하게 주입 / 실제 채용공고 크롤링 (v2) |
| 사용자가 답변을 "정답 베끼기" 용도로 오용 | 피드백은 제공하되, 모범답안 전체 생성은 의도적으로 제한 |
| AI 운영 비용이 사용량에 비례해 증가 | 무료 플랜은 1일 세션 수 제한 (예: 1일 3세션) |
| 자소서 등 민감 정보 유출 우려 | 암호화 저장 + 외부 학습 차단 명시 (개인정보 처리방침에 고지) |

## 부록 A. 용어 정의

- **세션:** 한 번의 면접 연습 단위 (질문 생성부터 종료까지)
- **STAR 기법:** Situation–Task–Action–Result. 면접 답변 구조화 기법
- **약점 요약:** 누적 피드백에서 반복되는 부정적 패턴을 한 줄로 추출한 것
